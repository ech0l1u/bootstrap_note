栅格系统

bootstrap提供一套响应式 我猜大概就是会随着浏览器窗口宽度自动改变排列的，移动设备优先的流式 就是铺满地，像 .container-fluid这样的栅格系统，最多12列

概述

栅格系统是用一系列行（ row）和列（ column）的组合来创建页面布局，注意是整个页面布局。创造好的布局可以塞在里面。其实栅格系统有点像用 <table>来创建布局，不同的是，bootsrap的栅格全是用 <div class = "row" >当成 <th>，用 <div class = "col-xs-N" >当成 <td>。前面那个 <div class = "col-xs-N" >里的 N我猜应该是可以是1到12之间的整数，但是最好还是12的约数会好一点吧。
突然想起来一种类名的写法，就是 class = "name1 name2"

虽然说类名里带 row和 col，但是实际上还是和 <th>和 <td>一样一行一行摞出来的。

.row必需位于 .container或者 .container-fluid里，内容放 .col-xs-N里，而且只有 .col-xs-N能当 .row的直接子元素

可以给 .col-xs-N设置 padding值使 col分隔开，可以给 .row设置负的 margin值来弥补 .container的 padding值，这样会间接为行（ row）所包含的列（ column）抵消掉了 padding 这个间接抵消是个啥意思啊？。

如果一行（ row）里列（ column）的数量超过12，超过的部分会另起一行显示，但是还是属于之前那一行（ row）的一部分

媒体查询虽然我看CSS时没有认真看媒体查询

在 bootstrap.css里，媒体查询有三个点，分别是768px，992px，1200px。这三个点就是媒体查询的分界点，在原生的bootstrap里，和屏幕宽度有关的媒体查询在原生的bootstrap的CSS里是这样的： 

@media (min-width: 768px) {
    .container{
        width: 750px;
    }
}
@media (min-width: 992px) {
    .container{
        width: 970px;
    }
}
@media (min-width: 1200px) {
    .container{
        width: 1170px;
    }
}

流式布局

前面说了流式布局就是栅格放在 .container-fluid里头的，因为现在这个位置最外面有一层 .container， .container-fluid和 .container不能相互嵌套 .container-fluid在 .container里的话也会有 margin这类东西的，所以就不展示了。

列偏移

使用 .col-md-offset-N 里面那个 md变成 sx sm lg应该也是可以的，N是整数可以往右偏移N列宽度，一行12列。 不知道超出去的部分在bootstrap里会怎么样啊

注意 .col-md-offset-N这个类是因为 .col-md-N才有意义的，所以要用别忘了最重要的 .col-md-N

---->
col-md-offset-3
----> col-md-offset-1
col-md-offset-4

通过实验发现，列偏移根本不会移出 .container外边去，因为要出去的都会出现在另外一行，有点像 float : "xxx"了。
